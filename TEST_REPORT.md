## 六、结论

### 6.1 成功指标

- [x] 全部 6 项测试通过，通过率 100%
- [x] 查询改写功能正常，意图识别准确

- [x] Pipeline 端到端流程跑通，答案质量良好
- [x] 新增功能开销可控（约增加28%耗时）
- [x] 向后兼容，原有功能不受影响

### 6.2 建议

1. **查询改写优化**：可针对年报场景进一步优化意图分类关键词库
2. **缓存机制**：对相同/相似查询的改写结果进行缓存，减少重复API调用
3. **参数调优**：可根据生产环境QPS限制调整并行请求数
4. **批量处理**：process_all_questions 可批量利用改写和重排序能力

### 6.3 风险点

- 查询改写依赖 DashScope API，网络波动可能增加延迟
- 增强重排序的 rerank API 在大量并发时可能触发限流
- 新增功能增加了约1.7秒延迟，在对实时性要求极高的场景需权衡
